## Risks + Mitigations

| Risk | Mitigation |
|---|---|
| RTL/jsdom setup breaks node-environment tests | Use `environmentMatchGlobs` in vitest.config.ts to scope jsdom only to `*.tsx` test files |
| Server action mocking is brittle (Resend/Slack/Meta CAPI) | Mock at `lib/` import boundary, not the server action itself; keep mocks minimal |
| `next/navigation` and `next-intl` may need polyfills in jsdom | Mock both via `vi.mock('@/i18n/navigation', ...)` if needed |
| Test runtime balloons | Target each component test at <500ms; if total runtime exceeds 5s, split happy-path from edge-cases into separate files |
